E1: Tweet

Fire up rails console. Then do the following things:

  • Create a tweet.
  • Find it with the Tweet.find method
  • Find it by user with Tweet.where(:user => "...")
  • Find it by user with Tweet.find_by(:user => "..."
  • Describe to yourself what the difference is between those two methods.
  • Update that tweet to have a different tweeter name. Is this possible?
  • Update that tweet to have a different body. Is this possible?
  • Update that tweet to have a different id. Is this possible?
  • Delete that tweet

Fueling Consumerism on Rails

We're going to begin the process of translating the project from last week into a Rails application. Each class (Item, Cart, Shopper...) will become its own model, and you'll write the controllers and views for them in time. This will also be a chance to go through the flow of building your own Rails application from scratch, as opposed to a codealong.

In your projects directory, create a new Rails app called shopping (Hint: rails new shopping). Make sure you can get the server running; you may need to kill other servers that were previously running on your Terminal.

As with before, we'll start with the Item class. Use scaffolding to generate an Item model. It should have a property name, which is a string, and a property price, which is a float. Once you've created it, verify it's working by visiting localhost:3000/items and creating, viewing and deleting a new item. What happens if you type a non-float in the price field? Can you figure out how to fix that?

Pretty 'n' Sassy

Sass up your webpage! Use SASS CSS in your app/assets/stylesheets/items.css.scss to make your website pretty (the .scss extension means you can use SASS, much like how .erb means you can use Ruby in HTML). Remember to add styles that apply to all pages (like styles on body or all text, for example) in application.css (which you can rename application.css.scss) and only those that apply to the items page in items.css.scss.
