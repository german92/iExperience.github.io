Exercises

E1: CRUDdy Review

  1. Map CRUD to HTTP methods, for example index is GET (y’all should probably have these down by now)
  2. For an application containing an author, authorship, and books, list all the relations that exist.
  3. A certain route.rb file has a get products/:id line. What can be an example to the corresponding URL issued by the browser? What params are set?
  4. What HTTP method does a form submission use by default?
  5. What is the difference between an edit and update action? Similarly, what is the difference between new and create?

E2: I Don't Want Any Part of This

Create a delete button on the user account that works. As an added bonus, make the button ask for confirmation before it deleted the account. Hint: the scaffolding code we created earlier in the course does this—checking it might provide clues on how to do this.

Also, try adding an edit button to be able to edit a user.

Not So Easy to Leave

As you may note, once you delete a user, your application will start having errors, complaining that now there are some lonely tweets. Make it so when a user gets deleted, all his/her tweets are deleted as well. This might prove useful. Hint: your answer should require less than one line of code.

E3: A Pretty Date ;)

The "created at 2014-06-05 08:29:04 UTC" is pretty ugly. Research the strftime method to make your date nicer to look at.

E4: Catch up

Since most of you have been trying to finish up on previous exercises, we’ve decided to give you some time to finish and ask questions. It is VV important to finish these exercises, so please get them done while we are around and can help you debug!
