Exercises

E1: Review

Answer the questions below to recall some of the concepts learned in class today. Make sure you know the answers to each before moving on to the next exercises.

  • What does "fat model, skinny controller" mean and what is its significance in rails development? This is an important concept that most beginners fail to implement.
  • What is the use of instance variables in controllers?
  • What's a short cut to generating routes?
  • What series of steps take place in the MVC framework when a browser issues the URL /users? Draw a relationship diagram showing the MVC structure and how it interacts to render /users URL in a browser. Number the steps. You can draw on paper but using a drawing tool is highly recommended.
  • If you edit your data model, what rails command do you issue to reflect that change on the database? (Buzz question)

E3: Guess what? Reweet

We previously created tweet tweet data, Now let's try writing controller logic to manipulate this tweet data.

  • Write a new method, piglatify_tweet that updates the Tweet to be in Piglatin. Make a route, a controller method and a button in the view so that with a click of a button, the user can turn one of his/her tweets into piglatin.
