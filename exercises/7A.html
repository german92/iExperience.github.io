E1: Review

Answer the questions below to recall some of the concepts learned in class today. Make sure you know the answers to each before moving on to the next exercises.

  1. What does "fat model, skinny controller" mean, and what is its significance in Rails development?
  2. Why do we use instance variables in controllers? That is, why is there an @ in front of some of them?
  3. What series of steps take place in the MVC framework when a browser issues the URL /users? Draw a relationship diagram showing the MVC structure and how it interacts to render /users URL in a browser. Number the steps.

{% if page.answers_shown %} Hover for answer
  1. Most of your logic should go in your model files. You controllers should only call methods that have been defined in the model.
  2. So that those variables can be accessible in the view. The view cannot see local variables created in the controller.
  3. Draw a diagram, but the flow should look like: Client -> Routes -> UsersController -> User Model -> Database -> Back to User Model -> Back to UsersController -> User index view -> Back to Client (a total of 9 steps)
{% endif %}
